## Blocked Models

### BaseBoosting

BaseBoosting uses the `olorenchemengine` (OCE) library, which is not available as a prebuilt wheel on PyPI. Installation requires cloning from GitHub (`github.com/Oloren-AI/olorenchemengine`). All three install methods were attempted and failed:

- `pip install olorenchemengine` — not available as a prebuilt wheel
- `pip install olorenchemengine -i <Tsinghua mirror>` — not on mirror
- `pip install git+https://github.com/Oloren-AI/...` — GitHub HTTPS blocked by company firewall

Additionally, even if OCE could be installed, `OlorenCheckpoint("default")` (the third learner in the ensemble) downloads a pre-trained GIN from `storage.googleapis.com`, which is also likely blocked.

The benchmark code is fully implemented in `model_assets/BaseBoosting/code/`. BaseBoosting can be run on any machine with unrestricted internet access to GitHub.

### ZairaChem

ZairaChem relies on the **Ersilia Model Hub** to download molecular descriptors at runtime. The hub fetches pre-trained models from GitHub and DockerHub — both blocked on the company server. Key model weights for `eos2gw4` and `eos2db3` are stored in Git LFS, which is also blocked. Only `eos5axz` (deterministic Morgan fingerprints, no weights needed) could be fetched, which is insufficient to run the full pipeline.

Several source code bugs were patched during the attempt:
- `manifolds.py` — NoneType not iterable when descriptor data is None
- `pool.py` — UnboundLocalError when features are empty
- `table.py` — IndexError when tasks list is empty
- `training.py` — AttributeError when featurizer_ids not set

The patched code is in `model_assets/ZairaChem/code/`. ZairaChem can be run on a machine with unrestricted access to GitHub and DockerHub.

## Deviations from Original Pipelines

### MiniMol
- **Original:** Pre-trained GNN encoder → MLP predictor, fixed hyperparameters
- **Ours:** Pre-trained GNN encoder → Random Forest predictor, added 4-combo hyperparameter grid (n_estimators: 50/100/200, max_depth: None/10)
- **Known issue:** Pre-training data may overlap with TDC test molecules — data leakage flagged in 2026 critical assessment. Scores may be slightly inflated.

### DeepMol
- **Original:** Full AutoML across 140+ model/featurizer combinations including mol2vec and RDKit descriptors
- **Ours:** Reduced to 6 combos (2 Morgan fingerprint sizes × 3 models) due to dependency conflicts. Used deepmol 1.1.7 instead of 1.2.1.

### MapLight_GNN
- **Original:** ECFP + Avalon + ErG + RDKit 2D + GIN embeddings → CatBoost, default hyperparameters
- **Ours:** Same feature pipeline, added 4-combo CatBoost tuning grid (iterations: 500/1000/2000, lr: 0.03/0.1). Fixed molfeat API change (`ecfp` not `ecfp:4`). Added `get_global_used_names()` to handle molecules that break the desc2d featurizer across different seeds.
- **Note:** Passed all reproducibility checks in 2026 critical assessment — most trustworthy baseline.

### AttrMasking
- **Original:** DGL_GIN_AttrMasking via DeepPurpose, fixed hyperparameters (lr=0.001, epochs=50)
- **Ours:** Same encoder, added 2-combo tuning grid (lr: 1e-3/1e-4, epochs: 20/30). Removed `binary=` parameter (not in installed version). Pre-trained weights downloaded manually to `~/.dgl/` due to network restrictions. Monkey-patched DGL download to prevent re-downloading.

### AttentiveFP
- **Original:** DGL_AttentiveFP via DeepPurpose, default hyperparameters
- **Ours:** Same encoder, added 3-combo tuning grid (lr: 1e-3/5e-4/1e-4, epochs: 20/30/50). Trains from scratch — no pre-trained weights needed.

### BasicML
- **Original:** 31 RDKit descriptors, per-task model selection from candidate pool, pre-selected best model per task
- **Ours:** Same 31 descriptors and 10-model candidate pool. Re-selects best model each seed via RandomizedSearchCV (8 iterations, 3-fold CV) rather than using pre-selected pkl files.

### CaliciBoost
- **Original:** AutoGluon AutoML across 8 feature types, identifies PaDEL + XGBoost as best, Bayesian hyperparameter optimization
- **Ours:** Goes directly to PaDEL + XGBoost with a 4-combo grid around the paper's best hyperparameters. Feature selection via permutation importance only (paper also uses SHAP). Covers caco2_wang only.
